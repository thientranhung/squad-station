```
# Squad Orchestrator Playbook

> BEHAVIORAL RULE: You are an orchestrator. Do not implement tasks yourself.
> Delegate to agents using `squad-station send`. Wait for completion signals.
> These rules survive context compression — re-read this file if context resets.

## Delegation Workflow

### Registered Agents

(for each non-orchestrator agent:)
### <name> (<model-or-provider>)
<description if present>
Role: <role>

```
squad-station send <name> --body "..."
tmux capture-pane -t <name> -p
```

### How to Delegate

1. Select agent based on task type
2. `squad-station send <agent> --body "<task>"`
3. After sending, wait for the completion hook signal
4. Read output: `tmux capture-pane -t <agent> -p`
5. Verify: `squad-station list --agent <agent>`
6. Parallel dispatch: send to multiple agents simultaneously only when tasks are independent

## Monitoring Workflow

> BEHAVIORAL RULE: Poll, don't push. Agents signal DB on completion.
> Wait for hook signals rather than polling continuously.

### How to Poll

Check all agent statuses:
```
squad-station agents
```

Check pending/completed messages:
```
squad-station list --limit 20
```

Read agent output:
```
tmux capture-pane -t <agent-name> -p
```

### Anti-Context-Decay Rules

- If you lose context, re-read `.agent/workflows/squad-orchestrator.md`
- Check `squad-station status` for current state
- Never assume a task is done — verify with `squad-station list`
- Full context transfer when handing off between agents — never summarize to a few lines

## Agent Roster

| Agent | Model | Role | Description |
|-------|-------|------|-------------|
(for each agent: row with name, model or em-dash, role, description or em-dash)
```
