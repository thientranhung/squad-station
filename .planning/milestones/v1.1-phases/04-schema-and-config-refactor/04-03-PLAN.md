Note on `SELECT *` vs explicit column list: After migration 0003, `SELECT * FROM agents` returns
columns in this order: id, name, tool, role, command (still exists as a column — NOT DROPPED),
created_at, status, status_updated_at, model, description, current_task.
The old `command` column remains in the DB (SQLite DROP COLUMN is out of scope).
Add `pub command: Option<String>` to the Agent struct as a dead-code field to satisfy FromRow,
OR switch to an explicit SELECT that omits command. The cleanest approach: use explicit SELECT
listing only the columns we want. But that requires updating every query. Simplest: add
`#[allow(dead_code)] pub command: Option<String>` to Agent struct — the column still exists
(NOT NULL DEFAULT '' from original schema), but after migration it stays as existing data.

Wait — the original `command TEXT NOT NULL` column cannot be made nullable via ALTER TABLE.
After migration 0003, the `command` column still has NOT NULL constraint. New `INSERT OR IGNORE`
statements must NOT include `command` in the column list or they will fail.
BUT the original schema has `command TEXT NOT NULL` — if we omit it from INSERT, SQLite requires
a DEFAULT value. There is no DEFAULT on the command column (it was `TEXT NOT NULL` with no default).

Resolution: Migration 0003 must ADD a DEFAULT to the command column, or provide a workaround.
Add to migration 0003_v11.sql (Plan 02's migration file — add this before merge):
```sql
-- Make command column optional (tools infer their own command — CONF-03)
-- SQLite cannot alter column constraints directly; use CREATE TABLE + INSERT approach
-- OR: Accept that new INSERT OR IGNORE must include command = '' as placeholder
-- Simplest: new INSERT always passes '' for command (empty string placeholder)
```
Actually the simplest safe approach: keep passing an empty string `""` for `command` in the
INSERT in Plan 03's `insert_agent` implementation — the column still accepts NOT NULL empty string.
This is a legacy artifact. The column stays with its NOT NULL constraint, new code passes "".
Document with a comment: `// legacy column; value is empty string placeholder`.
Do NOT try to ALTER the constraint — that requires full table rebuild.
