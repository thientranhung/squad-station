<task type="auto">
  <name>Task 1: Create GitHub Actions release workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
Before writing the file, check whether sqlx offline metadata exists:
`ls .sqlx/ 2>/dev/null || ls sqlx-data.json 2>/dev/null`
If neither exists, the build step must include `SQLX_OFFLINE: true` as an environment variable.

Create `.github/workflows/release.yml` with the following exact structure:

**Trigger:** `on: push: tags: ['v*']` — fires only on version tags, not on branch pushes.

**Permissions:** Set `contents: write` at the job level so `softprops/action-gh-release` can create releases and upload assets.

**Jobs — use a matrix strategy with `strategy: fail-fast: false` and 4 entries:**

```
include:
  - target: aarch64-apple-darwin
    os: macos-latest
    asset_name: squad-station-darwin-arm64
    use_cross: false

  - target: x86_64-apple-darwin
    os: macos-latest
    asset_name: squad-station-darwin-x86_64
    use_cross: false

  - target: x86_64-unknown-linux-musl
    os: ubuntu-latest
    asset_name: squad-station-linux-x86_64
    use_cross: false

  - target: aarch64-unknown-linux-musl
    os: ubuntu-latest
    asset_name: squad-station-linux-arm64
    use_cross: true
```

**Steps per matrix job:**

1. `actions/checkout@v4`

2. Install Rust stable: `dtolnay/rust-toolchain@stable` with `targets: ${{ matrix.target }}`

3. Conditional install of `cross` (only when `use_cross == true`):
   ```yaml
   - name: Install cross
     if: matrix.use_cross == true
     run: cargo install cross --git https://github.com/cross-rs/cross
   ```

4. Build step — use `cross` or `cargo` based on `use_cross`:
   ```yaml
   - name: Build
     env:
       SQLX_OFFLINE: "true"
     run: |
       if [ "${{ matrix.use_cross }}" = "true" ]; then
         cross build --release --target ${{ matrix.target }}
       else
         cargo build --release --target ${{ matrix.target }}
       fi
   ```

5. Rename binary to asset name for upload:
   ```yaml
   - name: Rename binary
     run: cp target/${{ matrix.target }}/release/squad-station ${{ matrix.asset_name }}
   ```

6. Upload asset and create/update GitHub Release:
   ```yaml
   - name: Upload to GitHub Release
     uses: softprops/action-gh-release@v2
     with:
       files: ${{ matrix.asset_name }}
       generate_release_notes: true
   ```

**Important implementation notes:**
- Use `musl` targets for Linux (not `gnu`) — produces fully static binaries with no glibc dependency. This is critical for Phase 9 install script to work on any Linux distribution.
- Both macOS targets run natively on `macos-latest`. The `dtolnay/rust-toolchain@stable` action handles `rustup target add` automatically when the `targets` field is set.
- Do NOT add a separate job to create the release. `softprops/action-gh-release` creates the release if it does not exist and appends assets to it if it does. All 4 matrix jobs safely upload to the same release identified by the git tag.
- `SQLX_OFFLINE: "true"` must be set on the build step so sqlx skips compile-time database connection checks (no DATABASE_URL available in CI).
- `fail-fast: false` on the matrix ensures all 4 targets are attempted even if one fails — useful for debugging partial failures.
