    1. Load config and connect to DB (same as current — keep these lines unchanged).

    2. Fetch agents via `db::agents::list_agents(&pool).await?` — DO NOT call `tmux::session_exists` or update agent status. Remove the reconciliation loop entirely. context is read-only with respect to DB state.

       ```
       # Squad Delegation Workflow

       > BEHAVIORAL RULE: You are an orchestrator. Do not implement tasks yourself.
       > Delegate to agents using `squad-station send`. Poll for completion.
       > These rules survive context compression — re-read this file if context resets.

       ## Registered Agents

       For each agent (role != "orchestrator"):
         ### <agent.name> (<model or tool>)
         <description if present>
         Role: <role>

         ```
         squad-station send <agent.name> --body "..."
         tmux capture-pane -t <agent.name> -p
         ```

       ## How to Delegate

       1. Select agent based on task type
       2. `squad-station send <agent> --body "<task>"`
       3. Poll: `squad-station agents` to check status
       4. Read output: `tmux capture-pane -t <agent> -p`
       5. Completion: `squad-station list --agent <agent>` to verify
       ```
       Write to: `std::fs::write(".agent/workflows/squad-delegate.md", content)?`

    5. Write squad-monitor.md via a `build_monitor_md() -> String` helper (static content):
       ```
       # Squad Monitor Workflow

       > BEHAVIORAL RULE: Poll, don't push. Agents signal DB on completion.
       > You do not receive push notifications if using antigravity provider.

       ## How to Poll

       Check all agent statuses:
       ```
       squad-station agents
       ```

       Check pending/completed messages:
       ```
       squad-station list --limit 20
       ```

       Read agent output:
       ```
       tmux capture-pane -t <agent-name> -p
       ```

       ## Anti-Context-Decay Rules

       - If you lose context, re-read `.agent/workflows/` files
       - Check `squad-station status` for current state
       - Never assume a task is done — verify with `squad-station list`
       - Re-read `.agent/workflows/squad-roster.md` to confirm agent names
       ```
       Write to: `std::fs::write(".agent/workflows/squad-monitor.md", content)?`

    6. Write squad-roster.md via a `build_roster_md(agents: &[Agent]) -> String` helper:

       ```
       |-------|-------|------|-------------|
       For each agent: | <name> | <model or "—"> | <role> | <description or "—"> |
       ```
       Write to: `std::fs::write(".agent/workflows/squad-roster.md", content)?`
