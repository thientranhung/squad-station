**1. Title and tagline**
```markdown
# Squad Station

Message routing and orchestration for AI agent squads — stateless CLI, no daemon.
```

**2. Brief description (2-3 sentences)**
What Squad Station does: routes messages between an AI orchestrator and N agents running in tmux sessions. Provider-agnostic (Claude Code, Gemini CLI, etc.). Each project stores state in its own SQLite DB at `~/.agentic-squad/<project>/station.db`.

**3. Installation section (heading: `## Installation`)**

Three sub-sections:

**3a. npm (recommended)**
```markdown
### npm (recommended)
npm install -g squad-station
```
Add a one-liner: "Requires Node.js 14+. Postinstall downloads the native binary for your platform."

**3b. curl (no Node.js required)**
```markdown
### curl
curl -fsSL https://raw.githubusercontent.com/thientranhung/squad-station/main/install.sh | sh
```
Add: "Installs to `/usr/local/bin` (falls back to `~/.local/bin`). Supports macOS and Linux."

**3c. Build from source**
```markdown
### Build from source
git clone https://github.com/thientranhung/squad-station.git
cd squad-station
cargo build --release
# Binary: target/release/squad-station
```
Add: "Requires Rust toolchain. See [Cargo docs](https://doc.rust-lang.org/cargo/getting-started/installation.html)."

**4. Quickstart section (heading: `## Quickstart`)**

Show three steps:

Step 1 — Create `squad.yml`:
```yaml
project: my-app
orchestrator:
  tool: claude-code
  role: orchestrator
agents:
  - name: backend
    tool: gemini
    role: worker
```

Step 2 — Initialize:
```bash
squad-station init
```
Caption: "Registers agents from squad.yml and opens their tmux sessions."

Step 3 — Send a task:
```bash
squad-station send --agent my-app-gemini-backend --task "Implement the /api/health endpoint"
```

Step 4 — Signal completion (run from inside the agent's tmux session or hook):
```bash
squad-station signal --agent my-app-gemini-backend
```

Step 5 — List pending tasks:
```bash
squad-station list
```

Add: "See [docs/PLAYBOOK.md](docs/PLAYBOOK.md) for the complete workflow guide."

**5. Architecture overview (heading: `## Architecture`)**

A short paragraph + bullet list:

Paragraph: Squad Station is a stateless Rust CLI. There is no background daemon. Every command opens the SQLite DB, reads or writes, and exits.

Bullets:
- `agents` table — registered agents with provider, role, command, status
- `messages` table — tasks routed to agents, with priority (urgent > high > normal) and status lifecycle
- tmux sessions — each agent runs in its own named session; `send-keys -l` prevents shell injection
- Hooks in `hooks/` detect task completion per provider and call `squad-station signal`

```
Requires: tmux, macOS or Linux (Windows not supported — tmux unavailable).
```
