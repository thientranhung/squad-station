  <action>
Create `install.sh` as a POSIX sh script (shebang `#!/bin/sh`, `set -e`). The script must:

1. **Set constants at top:**
   ```sh
   REPO="thientranhung/squad-station"
   VERSION="0.1.0"
   ```
   VERSION must match `package.json` version exactly.

2. **Platform detection using `uname`:**
   ```sh
   OS=$(uname -s | tr '[:upper:]' '[:lower:]')
   ARCH=$(uname -m)
   ```
   - Map `OS`: `darwin` stays `darwin`, `linux` stays `linux`. Anything else: print error + exit 1.
   - Map `ARCH`: `x86_64` stays `x86_64`, `arm64` or `aarch64` maps to `arm64`. Anything else: print error + exit 1.

3. **Construct asset name and URL:**
   ```sh
   ASSET="squad-station-${OS}-${ARCH}"
   URL="https://github.com/${REPO}/releases/download/v${VERSION}/${ASSET}"
   ```

4. **Install directory resolution:**
   Try `/usr/local/bin` first. If not writable (test with `[ -w /usr/local/bin ]`), fall back to `$HOME/.local/bin`. Create fallback dir with `mkdir -p` if needed.

5. **Download:**
   Use `curl -fsSL --proto '=https' --tlsv1.2 -o "$TMPFILE" "$URL"`.
   Use a tempfile via `TMPFILE=$(mktemp)` with a trap to clean up on exit: `trap 'rm -f "$TMPFILE"' EXIT`.

6. **Install and verify:**
   ```sh
   mv "$TMPFILE" "${INSTALL_DIR}/squad-station"
   chmod 755 "${INSTALL_DIR}/squad-station"
   ```
   After install, verify binary is executable: `[ -x "${INSTALL_DIR}/squad-station" ]` — exit 1 with error if not.

7. **Success message:**
   Print the install path and instruct user to add to PATH if fallback dir was used:
   ```
   Installed squad-station to /path/to/bin/squad-station
   Run: squad-station --version
   ```
   If `~/.local/bin` was used, also print: `Add ~/.local/bin to your PATH if not already present.`

8. **Error messages:** All errors go to stderr (`>&2`). Include the GitHub Releases URL in the unsupported platform error so users can download manually.

After writing the file, make it executable: `chmod +x install.sh`
